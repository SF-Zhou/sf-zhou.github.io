不推荐使用 robin_hood

2021.01.11
SF-Zhou

之前一位朋友在项目中使用了 robin_hood::unordered_map，说效果不错，并推荐给了我。看了文档，它 API 与 std::unordered_map 完全一致，可以直接将 std::unordered_map 替换为 robin_hood::unordered_map，就可以获得免费的性能提升，听起来非常不错了。写了简单了测试，随机数据下读写确实可以获得 2 到 3 倍的性能提升，然后就大胆地在项目里用上了，没想到还是 naive 了。

在 robin_hood 最新的版本 3.10.0 中，以下代码 100% 触发 std::overflow 异常。这对工业级应用来说是不可接受的。
